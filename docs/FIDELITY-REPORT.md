
# Alignment & Fidelity Report

## 1. Verdict

sentinel-harness is a genuinely well-built, honest Layer-1 reference wrapper whose *core library* tracks the real AgentCore Harness API with unusual precision — but whose *docs, config artifacts, and badges* promise a full three-layer platform the repo does not ship, and whose flagship "declarative config" path is dead on arrival.

- **(a) Fidelity to what AgentCore Harness actually does: ~80%.** The `core.py` primitives are near-exact: GA list-shape `systemPrompt` (`[{"text":...}]`), memory passed to `create` without the `optionalValue` wrapper, `runtimeSessionId` ≥ 33 chars, correct tool-config envelopes (`remoteMcp`/`agentCoreGateway`/`inlineFunction`/`agentcore_code_interpreter`), SSE streaming parse, cascade-delete semantics, `actorId` at invoke time, and the double-IAM caller model. The deductions are real API-fidelity defects, not cosmetics: the HITL contract is only half-implemented (pause detected, no two-message `toolResult` resume — so the central "human kills hallucination" loop never actually closes in code or evidence), `byo_memory` invents a `messagesCount` field, `allowedTools` in the shipped YAMLs use the wrong `gateway___tool` grammar instead of `@gateway/tool` (silently scoping nothing), `MODEL_HAIKU` mixes a `global.` prefix with a dated id, and the sample IAM policy grants `bedrock-agentcore:*` — which includes `InvokeAgentRuntimeCommand`, the exact action that bypasses `allowedTools` entirely (a security footgun the docs elsewhere warn against).
- **(b) Coverage of the customer's intended 3-layer SecOps architecture: ~35%.** Layer 1 is partially real and live-run (3 scenarios: CVE-triage, detection-gen, multi-harness parallel with a measured 2.64x speedup). Layers 2 (simulation / BAS long-running Runtime) and 3 (Agent Factory, sandbox PreToolUse hooks, Gateway registry, LiteLLM specialists, CDK stacks) ship **no runnable code at all** — only prose. Of the three flagship supervisors, none is ever created/invoked end-to-end; their YAMLs aren't loadable and reference Gateway tools (`search_registry`, `siem_query`, `enrich_ioc`, `contain_action`, etc.) that have no handler. The named "research-supervisor with specialist delegation" is only approximated by the generic 3-role demo; alert-triage has zero evidence. ARCHITECTURE.md is honest ("L2/L3 designed and partially built"); README's symmetric three-layer table and badge are not.

## 2. What it gets right

- **Core lifecycle is library-grade and API-accurate:** `create_harness`/`wait_ready`/`invoke`/`delete`/`cleanup` plus builders for code_interpreter, remote_mcp, gateway, inline_function, managed/BYO memory — all returning the exact envelopes the tests and scenarios expect.
- **Verified schema fidelity:** systemPrompt LIST shape, memory without `optionalValue` on create, session ≥ 33 chars (tested), no-hyphen name regex enforced everywhere, token-vault `${arn:...}` header interpolation preserved, `actorId` at invoke time, cascade-delete-by-default with `keep_memory` → `deleteManagedMemory:false`.
- **Honest about real limits (approvingly):** single-harness = single-agent is stated plainly and the multi-harness+supervisor workaround is the sanctioned pattern; async long-term memory latency (minutes-scale teach→wait→recall) is documented as expected, not hidden; ARCHITECTURE.md explicitly labels L2/L3 as designed-not-fully-built.
- **Live-validated L1 substance:** multi_harness (2.64x speedup) and cve_triage (code_interpreter fired for deterministic CVSS math + inline HITL gate + managed SEMANTIC/SUMMARIZATION memory with per-analyst `actorId`) are real, evidence-backed runs.
- **Hermetic tests pass green offline:** 42 passed, zero AWS calls (monkeypatched control plane), covering name regex, systemPrompt shape, session length, and every builder envelope.
- **Deterministic, offline-safe tooling:** `sigma_yara_lint` is a real LLM-free linter; all 5 tool handlers return well-formed stub contracts and gate live egress behind explicit env flags; `web_search` is text-only with a 2MB cap.
- **Clean anonymization + CI gate:** no real account IDs/keys/customer name; the CI secret-and-name scan is correctly self-non-matching and fails the build on any hit. LICENSE (MIT-0) matches pyproject; `.gitignore` is thorough.

## 4. Overclaims to fix

- **HARNESSES.md: "the loader injects the matching tool definition at create time" / "declarative config consumed by `create_harness()`."** No loader consumes `harness.yaml`; `create_harness` is only ever called with Python args from scenarios. → Correction: "harness.yaml files are illustrative reference configs; they are not consumed by the shipped CLI. See scenarios/ for the actual create_harness calls." (Or implement the loader.)
- **core.py docstring + README: inline_function gate "pauses the loop and hands control back to you (approve/reject)."** Only the pause is detected; no `toolResult` is ever sent. → Correction: "demonstrates the pause (stop_reason=tool_use) half of the HITL contract; resume-with-decision is not yet implemented."
- **SETUP.md: "Deny sts:AssumeRole … if you don't need role switching," while the sample grants `bedrock-agentcore:*` on `*`.** The sample is broader than the posture the prose advocates and does not exclude `InvokeAgentRuntimeCommand`. → Correction: ship an explicit least-privilege action list; add "allowedTools cannot restrict InvokeAgentRuntimeCommand — the only control is not granting the IAM action."
- **byo_memory `messagesCount` presented as a real BYO knob.** Not a documented field. → Correction: remove it; document `retrievalConfig` and truncation `slidingWindow numMessages` separately.
- **ARCHITECTURE.md: runtimes "run in a VPC with no public network mode; NAT egress allowlisted," egress via a `web_search` tool.** No VPC/network config, no Runtime, no wired web_search tool ship. → Correction: mark as design narrative under a "designed, not built" heading.
- **README badge "scenarios: live validated" + symmetric three-layer table.** Only 3 L1 scenarios ran live; L2/L3 ship no runnable code; named research-supervisor/alert-triage with specialist delegation don't exist. → Correction: scope the badge to the three L1 scenarios; add a "validated vs designed vs missing" matrix mirroring ARCHITECTURE.md:81-87.
- **evidence/README.md marks `adversarial_review_ran: true` as a green check.** Captured reviewer output has no verdict (preamble only), `approved_signal:false`. → Correction: mark as "reviewer invoked but did not emit a verdict in this run" until re-run.
- **README "Repo layout" lists harnesses/ and tools/ as usable Layer-1 assets.** YAMLs aren't loadable; handlers aren't wired to any harness/gateway. → Correction: annotate both as "illustrative / not integrated."
